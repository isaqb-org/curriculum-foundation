[[LG-04-03]]
==== LZ 04-03 [ehemaliges LZ 3-3]: Notations-/Modellierungsmittel für Beschreibung von Softwarearchitektur erläutern und anwenden (R2-R3)

Softwarearchitekt:innen kennen mindestens folgende Diagramme zur Notation von Architektursichten:

* UML-Klassen-, Paket- und Komponentendiagramme (R2)
* UML-Verteilungs-, Sequenz- und Aktivitätsdiagramme (R2)
* Aufzählungen, z. B. nummerierte Listen oder Stichpunkte (R2)
* UML-Zustandsdiagramme (R3)
* ArchiMate (R3)
* SysML (R3)
* C4, siehe <<brownc4>> (R3)
* Entity-Relationship Diagramme, siehe <<erd>> (R3)
* Flussdiagramme (R3)
* Business-Process-Modelling-Notation (BPMN) (R3).

[[LG-04-03]]
==== LG 04-03 [previously LG 3-3]: Explain and Apply Notations/Models to Describe Software Architecture (R2-R3)

Software architects know at least the following diagrams to describe architectural views:

* UML class, package, and component diagrams (R2)
* UML deployment, sequence, and activity diagrams (R2)
* enumerations, e.g. numbered lists or bullet points (R2)
* UML state machine diagrams (R3)
* ArchiMate (R3)
* SysML (R3)
* C4, see <<brownc4>> (R3)
* Entity-relationship diagrams, see <<erd>> (R3)
* flow charts (R3)
* business-process-modeling-notation (BPMN) (R3).
